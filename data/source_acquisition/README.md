# Source Acquisition Batches

Эта папка хранит входные партии до scoring. Здесь нет API, логинов, cookies, токенов, скачивания видео или публикаций.

Главное правило: сначала собрать и проверить источники, потом запускать генератор, сначала читать `source_candidates_review.md`, и только после этого думать о сценариях.

## Папки

- `24-72h\` - свежие ранние вспышки, 8-15 кандидатов.
- `7d\` - рабочая недельная партия, 12-25 кандидатов.
- `30d\` - месячная карта рынка, 20-50 кандидатов.

## Шаблоны

- `24-72h\discovery_YYYY-MM-DD_24-72h.csv`
- `7d\discovery_YYYY-MM-DD_7d.csv`
- `30d\discovery_YYYY-MM-DD_30d.csv`

Перед сбором скопируйте нужный шаблон в файл с текущей датой, например:

```text
data\source_acquisition\24-72h\discovery_2026-07-12_24-72h.csv
```

## Что заполнять минимум

- `source_url` - публичная ссылка на short/reel/clip.
- `platform` - из разрешенного списка генератора.
- `published_at` - дата публикации; если точная дата не видна, дата сбора и пометка в `notes`.
- `title_or_caption` - видимый title/caption без копирования длинного текста.
- `views` - если видны.
- `likes/comments/shares/saves` - если видны, иначе `0` и пометка в `notes`.
- `topic_hint` - одна из рабочих тем: долги, договоры, ремонт, застройщик, возврат товара, маркетплейсы, наследство, алименты.
- `notes` - источник запроса, качество метрик, почему ролик похож на сигнал.

Фраза для неполных публичных метрик:

```text
public_search; views visible_on_page; likes/comments unknown; published_at approximate; query=...; why_signal=...
```
