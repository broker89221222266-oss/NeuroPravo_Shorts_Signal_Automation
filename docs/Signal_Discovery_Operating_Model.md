# Signal Discovery Operating Model

Дата фиксации: 2026-07-12

Цель проекта: не разово найти несколько роликов, а регулярно находить залетевшие короткие видео, проверять совпадение с НейроПраво и только потом делать свои текстовые сценарии.

## Главный конвейер

1. Собрать свежие короткие ролики за 24-72 часа, 7 дней и 30 дней.
2. Сохранить исходники в CSV: ссылка, платформа, дата, автор, caption/title, просмотры, реакции, тема, заметки по качеству метрик.
3. Рассчитать силу сигнала: свежесть, просмотры, скорость/отрыв если доступны, вовлечение, полнота метрик, совпадение с темами НейроПраво.
4. Выпустить `source_candidates_review.md`: Александр видит исходные ролики и причины выбора до сценариев.
5. Только после этого Окно 2 делает новый сценарий по механике, не копируя текст исходника.

## Частота обновления

- Daily scan: короткая партия за последние 24-72 часа, чтобы ловить ранние вспышки.
- Twice-weekly scan: нормальная партия за последние 7 дней.
- Weekly 30-day scan: большой пересмотр за 30 дней, чтобы видеть устойчиво залетевшие ролики.

Стартовый практический режим: 3 обновления в неделю. Сценарии делать только по кандидатам, которые прошли source review.

## Статусы кандидатов

`ЗАЛЕТЕВШИЙ-КАНДИДАТ`:
- ролик свежий, обычно до 30 дней;
- final_score проходит порог;
- neuropravo_fit_score не ниже минимального;
- есть видимый публичный сигнал: просмотры и/или реакции;
- есть понятная механика залета: конфликт, боль, ошибка, деньги, сроки, документы, переписка или бытовой риск.

`ПОТЕНЦИАЛЬНЫЙ-СИГНАЛ`:
- метрики неполные или дата приблизительная;
- тема и конфликт сильные;
- перед сценарием нужен ручной просмотр исходника.

`НЕ БРАТЬ`:
- слабая связь с НейроПраво;
- мало публичного сигнала;
- старый ролик без причины брать;
- риск копирования выше пользы;
- нет переносимой механики.

## Что обязательно показывать Александру

Каждая партия должна давать исходную выборку, а не только готовые сценарии:

- ссылка на исходный ролик;
- платформа;
- дата публикации или пометка, что дата приблизительная;
- просмотры;
- лайки, комментарии, shares, saves, если доступны;
- тема;
- почему ролик считается залетевшим или потенциальным сигналом;
- почему подходит или не подходит НейроПраво;
- какая механика залета;
- насколько сильно сценарий должен отличаться от исходника;
- решение: брать, подумать, отложить.

Основной файл проверки: `source_candidates_review.md`.

## Правило для Окон

Окно 1 отвечает за механизм нахождения:
- источники и импорт;
- свежие партии;
- scoring;
- source review;
- smoke/regression;
- качество данных.

Окно 2 отвечает за сценарии только после допуска:
- берет кандидата из source review;
- смотрит механику залета;
- пишет новый текст под НейроПраво;
- не копирует чужой текст, персонажа, последовательность фраз и вывод;
- указывает legal-safe границы.

Если нет `ЗАЛЕТЕВШИЙ-КАНДИДАТ` или вручную подтвержденного `ПОТЕНЦИАЛЬНЫЙ-СИГНАЛ`, Окно 2 не пишет сценарий.

## Насколько отличается наш сценарий

Разрешено брать:
- тип боли;
- конфликт;
- ритм hook;
- эмоциональную механику;
- формат поворота;
- причину, почему зритель досматривает.

Запрещено копировать:
- текст;
- порядок фраз;
- конкретного героя;
- уникальную историю без переработки;
- чужой вывод;
- чужой CTA.

Норма: сильная переработка. Наш сценарий должен быть новым материалом для Александра, где исходник служит сигналом рынка, а не шаблоном для копирования.
